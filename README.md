## คำอธิบายโครงสร้างโปรเจกต์

index.js คือไฟล์หลักที่สร้าง Express App, ตั้งค่า Port, นำเข้า Routesต่างๆ

package.json คือไฟล์ตั้งค่าโปรเจกต์เก็บข้อมูล, รายชื่อ, คำสั่งรัน

case1.png คือไฟล์รูปภาพทดสอบที่ 1: ดึงรายการนักศึกษาทั้งหมด

case2.png คือไฟล์รูปภาพทดสอบที่ 2: เพิ่มข้อมูลนักศึกษาใหม่

case3.png คือไฟล์รูปภาพทดสอบที่ 3: ดึงข้อมูลนักศึกษารายบุคคล

case4.png คือไฟล์รูปภาพทดสอบที่ 4: แก้ไขข้อมูลนักศึกษา

case5.png คือไฟล์รูปภาพทดสอบที่ 5: ลบข้อมูลนักศึกษา

case6.png คือไฟล์รูปภาพทดสอบที่ 6: ทดสอบกรณีไม่พบข้อมูล

README.md คือไฟล์อธิบายโครงสร้างเจกต์และตอบคำถาม

## หากมี Route เพิ่มขึ้นอีก 10 เส้นทาง โค้ดในไฟล์เดียวจะเกิดปัญหาใด

จะทำให้อ่านและทำความเข้าใจโค้ดได้ยาก ไฟล์มีบรรทัดเยอะเกินไปที่จะไล่ดูหรือค้นหาทำให้เกิดความยาก

ดูแลรักษาและแก้ไขยาก ถ้าแก้ไข Routeอาจกระทบกับ Routeอื่นได้

ทำงานร่วมกับคนในทีมลำบาก ถ้าแก้ไขไฟล์หลักพร้อมกันจะเกิดความขัดแย้งของโค้ดและทำให้เสียเวลา

และจะขัดต่อหลักการ Modular Designทำให้แยกเลเยอร์การทำงานไม่ได้
